# Security Policy

## Supported Versions

This project is maintained on a best-effort basis. Security updates and fixes may be provided when issues are identified.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately through the available contact methods listed on the website or GitHub profile.

Please do not disclose security vulnerabilities publicly until they have been reviewed.

When reporting an issue, include:

* A description of the vulnerability
* Steps to reproduce the issue
* Affected pages, files, or features
* Screenshots or proof of concept, if applicable

## Responsible Disclosure

Please act in good faith and avoid:

* Accessing data that is not intended for you
* Modifying content without authorization
* Interrupting service availability
* Attempting privilege escalation
* Conducting denial-of-service attacks

## Scope

This policy applies to:

* The website source code
* GitHub Pages deployment
* Website scripts and assets

## No Warranty

This project is provided "as is" without warranties of any kind. Security fixes may be implemented at the sole discretion of the project owner.
